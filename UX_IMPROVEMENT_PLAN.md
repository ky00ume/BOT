# 츄마고치 봇 UX 개선 기획서

> 이 문서는 향후 작업 지침서로 사용되는 상세 기획서입니다.
> 각 항목은 현재 문제 → 개선 방안 → 수정 대상 파일 → 우선순위 순으로 기술합니다.

---

## 1. 명령어 체계 정리 및 일관성 확보

### 현재 문제
- `/대화`, `/구매`, `/판매`, `/알바` 등 행동 계열 명령어가 일부는 `NPC이름 인자`를 받고, 일부는 독립적으로 동작하여 일관성이 없음.
- `/구매 몰 작은 가방` vs `/판매 작은 가방` — 구매는 NPC 이름 필수, 판매는 불필요. 사용자에게 혼란 야기.
- `/스토리수집`, `/낚시순위`, `/주간순위` 등 하위 명령어가 분산되어 있어 진입점이 불명확.
- 특수 NPC 관련 신규 명령어(`/특수키워드`, `/계약확인` 등)가 일반 명령어와 섞여 있음.
- 도움말(`/도움말`)에 명령어 설명이 너무 길게 나열되어 가독성이 낮음.

### 개선 방안
1. **명령어 카테고리화**: `/도움말 NPC`, `/도움말 전투`, `/도움말 생활` 등 카테고리별 세부 도움말 추가.
2. **NPC 명령어 통일**: `/대화`, `/알바`, `/구매`, `/선물`은 항상 `[NPC이름]`을 첫 인자로 받도록 통일.
3. **특수 NPC 명령어 분리**: `/특수` prefix로 그룹화 — `/특수대화`, `/특수계약`, `/특수버프` 등.
4. **판매 명령어 개선**: `/판매목록` 실행 시 인터랙티브 드롭다운(Select Menu) 제공.
5. **자동완성 안내**: 잘못된 명령어 입력 시 유사한 명령어 제안("혹시 `/대화`를 의도하셨나요?").

### 수정 대상 파일
- `main.py` — 명령어 라우팅 전면 정리
- `town_notice.py` — 도움말 분류 재구성
- `special_npc.py` — 특수 NPC 명령어 그룹화

### 우선순위
**높음** — 사용자 진입 장벽과 직결.

## 2. 인터랙티브 UI (discord.ui.View/Select) 활용 확대

### 현재 문제
- `/구매목록`은 텍스트 목록을 출력하고 `/구매 [NPC] [이름]`을 별도로 입력해야 함.
- 키워드 대화 버튼 시스템은 구현되어 있으나, 상점/판매/알바에는 미적용.
- `/선물 [NPC]` 입력 후 아이템 선택이 텍스트 기반이라 불편함.
- 전투 스킬 선택이 `/공격 [스킬명]` 텍스트 입력 방식으로, 버튼 UI가 없음.

### 개선 방안
1. **상점 UI**: `/구매목록 [NPC]` 실행 시 `SelectMenu`로 아이템 선택, 수량 입력 Modal 제공.
2. **판매 UI**: `/판매목록` 실행 시 인벤토리 아이템 드롭다운 선택 → 판매 확인 버튼.
3. **선물 UI**: `/선물 [NPC]` 실행 시 인벤토리의 선물 가능 아이템을 드롭다운으로 표시.
4. **전투 UI**: 전투 중 스킬 버튼 자동 생성 (`View`). `/공격`은 legacy 지원 유지.
5. **알바 UI**: `/알바` 실행 시 알바 가능 NPC 목록을 버튼으로 선택.
6. **인카운터 UI**: 특수 NPC 인카운터 시 대화/선물/계약 버튼을 자동으로 View에 포함.

### 수정 대상 파일
- `shop_ui.py` — 상점/판매 Select UI 구현
- `main.py` — 전투, 선물, 알바 명령어에 View 추가
- `special_npc.py` — 인카운터 View 클래스 추가
- `npc_conversation.py` — 기존 KeywordView 패턴 확장

### 우선순위
**높음** — 게임 편의성 핵심 개선.

## 3. 신규 플레이어 온보딩 (튜토리얼) 설계

### 현재 문제
- 첫 접속 시 자동으로 게임 진입하지만, 기본 시스템 안내가 없음.
- `/도움말` 출력이 너무 많아 읽기 어려움.
- `/상태`, `/인벤토리`, `/대화`, `/사냥` 등 핵심 명령어를 스스로 발견해야 함.
- 새 플레이어가 첫 접속 시 에너지 100%, 골드 500G만 있고 가이드가 없음.

### 개선 방안
1. **단계형 튜토리얼 시스템**: 최초 `/시작` 또는 첫 접속 시 튜토리얼 플래그 설정.
   - Step 1: 캐릭터 이름 등록 + 기본 상태 확인 (`/상태`)
   - Step 2: 대화 첫 경험 — 다몬에게 `/대화 다몬`
   - Step 3: 첫 알바 — `/알바 다몬`
   - Step 4: 첫 구매 — `/구매목록 몰`
   - Step 5: 첫 사냥 — `/사냥터` → `/사냥 [사냥터]`
2. **퀘스트 마커**: 튜토리얼 단계를 `/퀘스트`에 "튜토리얼" 카테고리로 표시.
3. **팁 시스템**: 명령어 실행 후 50% 확률로 관련 팁 1줄 표시 (튜토리얼 완료 후 비활성화).
4. **NPC 첫인사 강화**: 처음 만나는 NPC는 자신의 역할과 가능한 명령어를 안내.

### 수정 대상 파일
- `main.py` — 튜토리얼 플래그 관리, 첫 접속 감지
- `player.py` — tutorial_step 필드 추가
- `database.py` — tutorial_step 컬럼 추가
- `quest.py` — 튜토리얼 퀘스트 데이터 추가
- 신규 `tutorial.py` — 튜토리얼 단계별 로직

### 우선순위
**중간** — 장기 플레이어보다 신규 유입 시 중요.

## 4. 정보 표시 통일 (ANSI 블록 vs Embed 혼용 문제)

### 현재 문제
- `상태`, `전투 결과`, `구매 완료` 등은 Discord Embed 사용.
- `대화`, `구매목록`, `판매목록`, `알바` 등은 ANSI 코드블록 텍스트 사용.
- 모바일 Discord에서 ANSI 블록이 렌더링되지 않아 가독성 저하.
- 일관성 없는 표시로 UI 통일감 부재.

### 개선 방안
1. **Embed 우선 정책**: 중요 정보(상태창, 전투, 상점)는 Embed 사용. ANSI는 간단한 응답에만.
2. **모바일 폴백**: ANSI 블록이 실패할 경우를 대비한 일반 텍스트 폴백 구현.
3. **색상 테마 통일**: `ui_theme.py`의 `EMBED_COLOR` 딕셔너리를 확장하여 모든 Embed 색상 표준화.
   - `status` (파랑), `battle` (빨강), `shop` (초록), `npc` (보라), `encounter` (오렌지)
4. **아이콘 통일**: 각 카테고리별 고정 아이콘 사용 (대화: 💬, 상점: 🛒, 전투: ⚔, NPC: 👤).
5. **에러 메시지 Embed**: 오류 메시지도 Embed의 빨간색 필드로 통일.

### 수정 대상 파일
- `ui_theme.py` — EMBED_COLOR 확장, 공통 Embed 빌더 함수 추가
- `shop.py`, `npcs.py`, `npc_conversation.py` — Embed 전환
- `main.py` — 오류 메시지 Embed 전환

### 우선순위
**중간** — 모바일 사용자 경험 개선.

## 5. 인벤토리 / 장비 / 상태 UI 개선

### 현재 문제
- `/인벤토리` 출력이 텍스트 기반으로 아이템 수 많을 시 길어짐.
- 아이템 정렬 없이 획득 순서대로 나열.
- 가방(slot)이 `/상태`에 표시되지 않았음 (이번 작업으로 수정됨).
- `/장비` 명령어와 `/상태` 명령어가 분리되어 있어 한 눈에 파악 불가.
- 아이템 수량 표시가 일부 명령어에서 누락.

### 개선 방안
1. **카테고리별 정렬**: 인벤토리를 무기/방어구/소비/재료/기타로 분류하여 표시.
2. **통합 상태창**: `/상태` 명령어 실행 시 장비 정보를 탭(Tab) 스타일 버튼으로 전환 가능.
3. **아이템 상세 조회**: `/아이템 [이름]` 명령어로 아이템 설명, 판매가 등 확인.
4. **가방 상태 표시**: `/상태`에서 `🎒 대형 가방 (10+20=30칸 / 사용 15칸)` 형태로 표시 (현재 부분 구현).
5. **자동 정리**: `/인벤 정리` 명령어로 같은 아이템 수량 합산, 0개 항목 제거.
6. **페이지네이션**: 아이템이 많을 경우 Previous/Next 버튼으로 페이지 이동.

### 수정 대상 파일
- `status_window.py` — 통합 상태창 탭 구현
- `equipment_window.py` — 장비창 Embed 개선
- `main.py` — 인벤토리 명령어 개선, 아이템 상세 명령어 추가
- `player.py` — 인벤토리 정렬/정리 메서드 추가

### 우선순위
**중간** — 장기 플레이어 편의성에 중요.

## 6. NPC 상호작용 흐름 개선 (대화→키워드→선물→알바의 동선)

### 현재 문제
- `/대화 [NPC]` → 키워드 버튼 클릭 → 다시 새 창에서 응답 — View 업데이트가 불안정.
- 대화 후 선물이나 알바로 자연스럽게 이어지는 흐름이 없음.
- 키워드 버튼이 최대 25개 제한으로 키워드가 많을 때 일부 숨겨짐.
- NPC 대화 종료 조건이 없어 View가 만료(timeout)되면 갑자기 사라짐.
- 특수 NPC 인카운터 시 대화 가능 여부가 불명확.

### 개선 방안
1. **통합 NPC 패널**: `/대화 [NPC]` 실행 시 키워드, 선물, 알바 탭을 하나의 Embed+View로 통합.
2. **키워드 페이지네이션**: 키워드가 8개 초과 시 Previous/Next 버튼으로 페이지 이동.
3. **선물 바로가기**: 대화 View에 "🎁 선물하기" 버튼 추가 → 인벤토리 드롭다운 바로 연결.
4. **알바 바로가기**: 알바 가능 NPC의 대화 View에 "💼 알바하기" 버튼 추가.
5. **대화 종료 버튼**: "👋 작별인사" 버튼 추가 — 클릭 시 NPC가 마지막 인삿말을 하고 View 닫힘.
6. **인카운터 NPC 상태 표시**: 현재 활성 인카운터 NPC를 `/상태`에 표시 (예: "🔴 라파엘 인카운터 중").

### 수정 대상 파일
- `npc_conversation.py` — 통합 NPC 패널 View, 키워드 페이지네이션
- `main.py` — 대화 명령어에 통합 View 적용
- `special_npc.py` — 인카운터 패널 View
- `status_window.py` — 인카운터 상태 표시

### 우선순위
**높음** — NPC 상호작용이 게임의 핵심 콘텐츠.

## 7. 전투 시스템 UX (스킬 사용, 턴 표시, 보상 표시)

### 현재 문제
- 전투 중 사용 가능한 스킬이 `/사냥` 시 한 번 Embed에 표시되지만, 이후 턴에서는 어떤 스킬을 쓸 수 있는지 확인 불가.
- `/공격 [스킬명]`으로 텍스트 입력 — 스킬 이름을 기억해야 하는 불편함.
- 전투 승리/패배 후 보상 Embed가 간략하여 드롭 아이템 확인이 어려움.
- 도주(`/도주`) 성공 확률이 표시되지 않음.
- 전투 종료 후 자동 저장이 되지 않아 진행이 소실될 수 있음.

### 개선 방안
1. **전투 스킬 버튼**: `/사냥` 이후 매 턴마다 사용 가능한 스킬 버튼 View 자동 생성.
   - 스킬 이름 + MP 비용 표시 (예: `🗡 강타 | 💙 0MP`)
   - 버튼 비활성화: MP 부족 시 해당 스킬 버튼 disabled 처리.
2. **현황 패널 지속**: 매 턴마다 내 HP/MP와 몬스터 HP를 Embed 상단에 표시.
3. **보상 상세 표시**: 전투 승리 시 드롭 아이템을 개별 필드로 표시, 총 보상 강조.
4. **도주 확률 표시**: `/도주` 명령어 사용 시 성공 확률 및 결과 표시.
5. **전투 후 자동 저장**: 전투 종료(승리/패배/도주) 즉시 DB 저장.
6. **라파엘 계약 연동**: 전투 승리 후 계약 몬스터였을 경우 진행도 표시 (현재 부분 구현).

### 수정 대상 파일
- `battle.py` — 스킬 버튼 View, 자동 저장
- `main.py` — 전투 명령어 View 연결
- `skills_db.py` — 스킬 UI 메타데이터 추가 (아이콘 등)

### 우선순위
**중간** — 전투가 주요 콘텐츠이나 기본 흐름은 작동 중.

## 8. 에너지/피로도 시스템의 투명성 확보

### 현재 문제
- 각 행동(알바, 채집, 낚시, 사냥 등)의 에너지 비용이 명령어 실행 전에는 표시되지 않음.
- 에너지가 부족할 때 실패 메시지만 나오고, 회복 방법이 안내되지 않음.
- `/휴식` 명령어가 있지만 얼마나 회복되는지 미리 알 수 없음.
- 에너지 회복 속도(시간당 자동 회복)에 대한 정보가 없음.
- 최대 에너지가 레벨에 따라 변동되는지 명확하지 않음.

### 개선 방안
1. **행동 비용 사전 표시**: 각 행동 명령어 설명에 에너지 비용 명시.
   - 예: `/채집 — 채집 실행 (기력 15 소비)`
2. **에너지 부족 시 안내 개선**: 단순 오류 대신 "💚 기력이 부족합니다! `/휴식`으로 회복하거나 에너지 회복 아이템을 사용하세요." 안내.
3. **회복 미리보기**: `/휴식` 실행 전 예상 회복량 표시.
4. **에너지 바 개선**: `/상태`의 에너지 바에 "다음 행동 가능까지 X 기력 필요" 정보 추가.
5. **일일 리셋 안내**: 에너지가 매일 자정에 리셋되는 경우, 리셋 시간을 `/상태`에 표시.
6. **에너지 효율 팁**: 에너지 대비 보상이 좋은 행동을 `/도움말 에너지`에서 안내.

### 수정 대상 파일
- `main.py` — 각 행동 명령어 에러 메시지 개선
- `status_window.py` — 에너지 표시 개선
- `town_notice.py` — 도움말에 에너지 가이드 추가
- `rest.py` — 회복 미리보기 기능

### 우선순위
**중간** — 플레이어가 시스템을 이해하지 못해 이탈하는 원인.

## 9. 알림/피드백 메시지 일관성

### 현재 문제
- 성공 메시지, 오류 메시지, 안내 메시지의 형식이 통일되지 않음.
  - 일부는 `✔` 아이콘 사용, 일부는 없음.
  - 일부는 ANSI 색상, 일부는 일반 텍스트.
- 중요 이벤트(레벨업, 업적 달성, 인카운터)는 강조 표시가 필요하나 통일성 없음.
- 저장 성공/실패 피드백이 일부 경우에서 누락.
- 긴 작업(낚시 타이밍 게임 등) 중 대기 중임을 알리는 표시가 없음.

### 개선 방안
1. **메시지 템플릿 정의**: `ui_theme.py`에 성공/오류/안내/경고 메시지 헬퍼 함수 추가.
   ```python
   def msg_success(text): ...  # ✔ 초록색
   def msg_error(text): ...    # ✖ 빨간색
   def msg_info(text): ...     # ℹ 회색
   def msg_event(text): ...    # ✦ 노란색 (레벨업 등)
   ```
2. **중요 이벤트 강조**: 레벨업, 업적, 인카운터는 별도 Embed로 강조.
3. **저장 피드백**: `/저장` 명령어 외에도 중요 행동 후 자동 저장 시 "(자동 저장됨)" 표시.
4. **로딩 표시**: 긴 처리 중에는 Discord의 `typing()` 기능 활용.
5. **한국어 통일**: 모든 메시지가 "슴미댜" 어미로 끝나도록 통일 (현재 혼용).

### 수정 대상 파일
- `ui_theme.py` — 메시지 템플릿 헬퍼 추가
- `main.py`, `shop.py`, `battle.py` 등 전체 — 메시지 형식 통일
- `npcs.py`, `npc_conversation.py` — NPC 메시지 통일

### 우선순위
**낮음** — 기능보다는 polish 영역.

## 10. 게임 진행 지표 및 콘텐츠 탐색성 개선

### 현재 문제
- 현재 플레이어가 무엇을 해야 할지 알기 어려움 (퀘스트는 있지만 접근성 낮음).
- `/스토리수집` 같은 특수 명령어가 도움말에 숨겨져 있음.
- 일일 수행 가능한 활동들(알바, 낚시, 채집)의 일일 제한/리셋 정보가 없음.
- 호감도 시스템의 혜택(할인, 특별 대화 등)이 명확히 안내되지 않음.
- 특수 NPC 인카운터가 언제 올지 알 수 없어 기대감 형성이 어려움.

### 개선 방안
1. **일일 활동 요약**: `/오늘할일` 명령어 — 오늘 남은 알바 횟수, 선물 횟수, 에너지 현황을 한눈에.
2. **진행도 표시**: `/진행도` 명령어 — 스토리 퀘스트, 업적, 호감도 진행 현황 요약.
3. **인카운터 힌트**: 마지막 특수 NPC 인카운터로부터 X일 경과 시 "어딘가에서 기척이 느껴진다..." 같은 힌트 메시지.
4. **호감도 혜택 안내**: `/대화` 시 현재 호감도 단계에서 받을 수 있는 혜택 표시.
   - 예: "💖 지인 단계 — 상점 5% 할인 / [대장간] 키워드 해금"
5. **콘텐츠 달력**: `/이번주` 명령어 — 이번 주 낚시 순위, 시즌 이벤트, 특수 날씨 예보.
6. **라파엘 계약 알림**: 계약 완료 가능 상태일 때 다음 활동 후 자동 알림.

### 수정 대상 파일
- `main.py` — `/오늘할일`, `/진행도`, `/이번주` 명령어 추가
- `special_npc.py` — 인카운터 힌트 로직
- `affinity.py` — 혜택 정보 표시 메서드 추가
- 신규 `progress.py` — 진행도 관리

### 우선순위
**낮음** — 중급~고급 플레이어 유지에 중요하나 기본 시스템 이후 추가.
